🏠 Overview

▼

As System Admin, you have unrestricted access to every part of the Holy Dham CMS. You are the highest authority and can manage all content, users, settings, and system configuration.

What You See in the Sidebar

  • Dashboard — Overview of site stats and recent activity
  • Holy Dhams — Create, edit, delete all sacred destinations (T1–T4)
  • Change Approvals — Review and approve/reject changes submitted by editors and photographers
  • Module Builder — Create custom content modules (e.g., Festivals, Saints, Temples)
  • All Entries — View and manage entries across all modules
  • Tags & Categories — Organize content with tags and category groups
  • Itineraries — Create travel itineraries linking multiple Dhams
  • Media Library — Browse and manage all uploaded images
  • Custom Fields — Define additional data fields for tiers (only System Admin)
  • Users & Roles — Create users, assign roles, manage Dham assignments
  • Site Pages — Manage static pages (About, Contact, etc.)
  • Reports — View error/feedback reports submitted by visitors
  • Email Settings — Configure SMTP for sending emails
  • Help Guide — This page (with role switcher and comparison table)

🏛️ Holy Dhams Management

▼

Holy Dhams are the core content of the CMS, organized in a 4-tier hierarchy:

T1 — Dham
→
T2 — Key Place
→
T3 — Key Spot
→
T4 — Key Point

Tier 1 — Holy Dham (Main Sacred Destination)

The top-level entry. Example: Vrindavan Dham, Ayodhya Dham.

Fields: Title, Short Description, Full Content, Featured Image, Gallery, State, City, Dham Code, GPS Location, Status (draft/published), Hero Slider, Category, Tags.

Tier 2 — Key Places (Important Areas)

Major areas within a Dham. Example: ISKCON Vrindavan inside Vrindavan Dham.

Each T1 can have multiple T2 entries. Fields: Title, Short Description, Full Content, Featured Image, Gallery, GPS, Category.

Tier 3 — Key Spots (Specific Locations)

Specific spots within a Key Place. Example: Main Temple Hall inside ISKCON Vrindavan.

Tier 4 — Key Points (Sacred Micro-Points)

Exact sacred points. Example: Radha-Krishna Deities inside Main Temple Hall.

Hierarchy IDs

Every entry gets an auto-generated hierarchy ID like AYD-001-003-002 (Dham-T2-T3-T4) for easy identification and search.

Actions You Can Perform

  • Create new entries at any tier
  • Edit all fields including text, images, GPS, categories, tags
  • Delete any entry (with cascading deletion of children)
  • Publish / Unpublish entries
  • Changes go live immediately — no approval needed

📋 Change Approvals

▼

When editors or photographers make changes, their work goes to the Change Approvals queue instead of going live immediately.

How It Works

  • A red badge on "Change Approvals" in the sidebar shows the count of pending items
  • Click to see the list of pending changes
  • Each item shows: who submitted, what Dham, what tier, what changed, and when
  • Click on an item to see the full comparison — previous data vs. updated data, side by side
  • For image uploads, you'll see thumbnail previews of the uploaded images

Approve or Reject

  • Approve — Changes are applied to the live content immediately. Images are linked, text is updated.
  • Reject — Changes are discarded. Uploaded files are deleted from disk. Optionally add a review note.

Approved History

Switch to the "Approved" tab to see the full audit trail — what was changed, who made the change, who approved it, and when. This log is paginated for performance.

📐 Tier Structure & Fields

▼

Common Fields Across All Tiers

FieldDescription
TitleThe name of the entry. Used in URLs (slug), search, and navigation.
Short DescriptionBrief summary shown in listings and cards (2-3 sentences).
Full ContentRich text content with detailed information. Supports the rich text editor.

FieldDescription

StatusDraft or Published. Only published Dhams appear on the frontend.
Hero SliderToggle to include this Dham's image in the homepage hero slider.
GPS LocationLatitude/longitude via map picker, search, or GPS capture.

Custom Fields

As System Admin, you can create additional fields from Custom Fields in the sidebar. Choose field type (text, number, date, dropdown, etc.), which tier it applies to, and sort order. These fields automatically appear in the edit forms.

Location Picker

Three ways to set GPS coordinates: (1) Search for a place name, (2) Click on the map, (3) Capture current GPS on-site. Coordinates are stored as latitude/longitude.

Camera Capture

The camera button lets users take photos directly from their device camera. Images are automatically compressed to keep file sizes reasonable. Works on both mobile and desktop.

📦 Content Management

▼

Module Builder

Modules let you create custom content types beyond the Dham hierarchy. Examples: Festivals, Saints & Acharyas, Temples, Prayers. Each module has its own icon, slug, and entries.

All Entries

A unified view of entries across all modules. Filter by module using the sidebar sub-items.

Tags & Categories

Categories provide a structured tree for organizing content (e.g., "Temple Type" → "ISKCON", "Ancient").

Tags are flat keyword labels for flexible cross-referencing (e.g., "must-visit", "photography-spot").

Itineraries

Create travel itineraries that link multiple Dhams together in a recommended visiting order. Useful for pilgrimage routes.

Site Pages

Manage static pages like About, Contact, Disclaimer. Content is edited using the rich text editor.

👥 Users & Roles

▼

Creating a New User

Go to Users & Roles → Add New User. Enter the email and select a role. The system will:

  • Auto-generate a secure password
  • Send a verification email (requires SMTP configured in Email Settings)
  • After the user verifies their email, login credentials are sent
  • On first login, the user must change their password

Roles Explained

  • System Admin — Unrestricted. Can manage custom fields, all settings, everything.
  • Admin — Full content management. Can approve/reject changes, manage users. Cannot manage custom fields.
  • Editor — Assigned to specific Dham(s). Can edit content only for assigned Dhams. All changes go through approval.
  • Photographer — Can upload photos across all Dhams via the Photos page. Uploads go through approval. Cannot edit text content.

Dham Assignment (Editors)

When creating or editing an Editor, you'll see a Dham assignment section. Check which Dhams this editor can edit. They'll see all Dhams but can only modify their assigned ones — unassigned Dhams appear greyed out with "View Only" labels.

⚙️ System Settings

▼

Custom Fields (System Admin Only)

Define additional data fields that appear in edit forms. Options: text, textarea, number, date, dropdown, checkbox, URL, email. Each field can be assigned to specific tiers (T1, T2, T3, T4) or section-based groupings.

Email Settings

Configure SMTP for sending emails (verification, password reset, credentials). For Gmail: Host = smtp.gmail.com, Port = 587, use an App Password (not your regular password).

Reports

View error reports and feedback submitted by website visitors via the frontend contact/report form. Mark as read, respond, or delete.

Media Library

Browse all uploaded images across the site. Useful for finding and reusing existing images.

💡 Tips & Best Practices

▼
  • Content creation order: Create T1 (Dham) first, then T2, T3, T4 in sequence.
  • Always add a featured image — it appears in cards, hero sections, and social shares.
  • Use meaningful Dham Codes — AYD for Ayodhya, VRN for Vrindavan. These appear in hierarchy IDs.
  • Check the Approvals queue regularly — editors and photographers are waiting for your review.
  • Set up Email Settings first — before creating new users, so verification emails are sent.
  • Use the "View Website" link to preview how content looks on the live frontend.
  • Create categories before content — so editors can assign them while creating entries.

✏️ How Editing Works

▼

When you open an assigned Dham, you'll see an "Approval Mode" banner at the top. This means:

  • You edit fields just like normal — change text, upload images, add entries
  • The Save button says "Submit for Approval" instead of "Save"
  • When you submit, your changes do NOT go live immediately
  • Instead, they go to the admin's Change Approvals queue
  • An Admin will review your changes and either approve (goes live) or reject (discarded)

Editing T1 (Dham) Fields

Click the edit button on your assigned Dham. Update any text fields, upload a new featured image, add gallery photos. Click "Submit for Approval" when done.

Editing T2 / T3 / T4

Navigate into the Dham → click "Key Places" to see T2 entries. Click on a T2 to see its T3 "Key Spots", and so on. Each level has its own "Submit for Approval" button.

Creating New Entries

You can create new T2, T3, or T4 entries within your assigned Dhams. Fill in the fields and submit. The new entry will appear once approved by an Admin.

Field Reference

FieldWhat to Enter
TitleClear, concise name. This becomes the URL and appears in navigation.
Short Description2-3 sentence summary shown in listing cards.
Full ContentDetailed content. Use the rich text editor for formatting, headings, links.
Featured ImageMain image. Upload from device or capture with camera. Landscape orientation works best.
GalleryAdditional photos. Upload multiple at once.
CategorySelect from available categories (set up by Admin).
GPS LocationSearch for a place, click on map, or use GPS to capture your current location.

📋 The Approval Process

▼

Here's what happens after you submit:

  • Step 1: You submit changes → a confirmation message appears: "Your changes have been submitted for approval"
  • Step 2: The Admin sees your submission in their Change Approvals queue with a red badge
  • Step 3: The Admin reviews your changes — they can see exactly what you changed compared to the previous version
  • Step 4: The Admin either Approves (changes go live) or Rejects (changes are discarded)
✅ Tip: Make your changes thorough before submitting — each submission is a batch. It's better to make all related changes at once rather than submitting multiple small updates.

📸 Uploading Photos

▼

How to Upload

  • Go to Holy Dhams in the sidebar
  • Click the 📷 Photos button on any Dham
  • You'll see the Photos page with all tiers (T1, T2, T3, T4) of that Dham
  • Each tier shows its current featured image and gallery

Two Types of Images

  • Featured Image — The main image for an entry. Shown in cards, headers, and social shares. Upload from device or capture with camera.
  • Gallery Images — Additional photos. Multiple can be uploaded at once.

Upload Methods

  • 📁 Upload from device — Click "Choose File" to select from your phone gallery or computer
  • 📷 Camera capture — Click the camera button to take a photo directly. Great for on-site photography!

After selecting/capturing images, click "Submit Photos for Approval".

⚠️ You cannot edit text fields, delete images, or create new entries. If you see any text fields, they are read-only for your reference.

📷 Camera Capture Tips

▼
  • Use landscape orientation for featured images (hold phone sideways)
  • Ensure good lighting — early morning or late afternoon light is best for temple photography
  • Hold steady — avoid blurry photos
  • Capture the essence — include architecture, deities, gardens, and sacred elements
  • Images are auto-compressed — don't worry about file size, the system handles it

On-Site Photography Workflow

  • Open the Holy Dham CMS on your phone
  • Navigate to the relevant Dham → Photos page
  • Use the camera capture button for each tier you're photographing
  • Submit all photos at once when you're done with that Dham

📋 What Happens After Upload

▼
  • Step 1: You upload photos and click "Submit Photos for Approval"
  • Step 2: A confirmation message appears with the count of photos submitted
  • Step 3: An Admin sees your uploads in their Change Approvals queue
  • Step 4: The Admin reviews the photos with previews
  • Step 5: Approved — photos are linked to the entries and appear on the website. Rejected — photos are deleted with an optional reason.
✅ Tip: Upload all photos for a Dham in one session rather than one photo at a time. This makes it easier for Admins to review.

🖼️ Media Library

▼

The Media Library shows all images that have been uploaded and approved across the site. Use it to see which entries already have photos and which ones need new images.

💡 Tips for Photographers

▼
  • Featured images matter most — they're the face of each entry
  • Cover all tiers — don't just photograph the main Dham (T1). Capture Key Places (T2), Spots (T3), and Points (T4) too
  • Quantity and quality — upload multiple gallery photos to give visitors a rich visual experience
  • Check the live site — use "View Website" to see which entries look good and which need better photos
  • Landscape for featured, any for gallery — featured images work best in landscape orientation; gallery can be any

  • You'll see all tiers (T1, T2, T3, T4) with image upload areas
  • Featured Image — Main image for an entry. One per tier entry.
  • Gallery Images — Additional photos. Multiple per entry.
  • Upload from device or use camera capture
  • Click "Submit Photos for Approval" when done

📷 Camera Tips

▼
  • Use landscape orientation for featured images
  • Ensure good lighting — morning/evening light is best
  • Hold steady to avoid blur
  • Capture architecture, deities, gardens, and sacred elements
  • Images are auto-compressed — no need to worry about file size

📋 After You Upload

▼
  • 1. Submit photos → confirmation with count
  • 2. Admin sees uploads in their approval queue
  • 3. Admin reviews with previews
  • 4. Approved → photos appear on website. Rejected → deleted with reason.
✅ Upload all photos for a Dham in one session rather than one at a time.

🖼️ Media Library

▼

Browse all approved images across the site. Use it to see which entries have photos and which need new ones.

💡 Tips

▼
  • Featured images matter most — they're the face of each entry
  • Cover all tiers — T1 through T4
  • Upload multiple gallery photos for a rich experience
  • Check "View Website" to see what needs better photos
  • Landscape for featured, any orientation for gallery
